# Markdown Cheat Sheet

A practical Markdown cheat sheet with examples for GitHub, Obsidian, Notion, VS Code, documentation projects, and README files.

Markdown is a lightweight markup language used for documentation, blogs, notes, GitHub repositories, and technical writing. This cheat sheet covers the most commonly used Markdown syntax with simple examples.

## Headings

```md
# Heading 1
## Heading 2
### Heading 3
#### Heading 4
```

## Text Formatting

```md
**Bold Text**

*Italic Text*

***Bold and Italic***

~~Strikethrough~~
```

## Lists

### Unordered List

```md
- Item One
- Item Two
- Item Three
```

### Ordered List

```md
1. First Item
2. Second Item
3. Third Item
```

## Links

```md
[GitHub](https://github.com)
```

## Blockquotes

```md
> This is a blockquote example.
```

## Inline Code

```md
`npm install`
```

## Code Blocks

````md
```javascript
function hello() {
  console.log("Hello World");
}
```
````

## Common Markdown Use Cases

### GitHub README Files

Create project documentation and README files.

### Technical Documentation

Write software documentation, API guides, and knowledge bases.

### Note Taking

Use Markdown in apps like Obsidian and Logseq.

### Blog Writing

Publish content with Markdown-supported CMS and static site generators.

### Team Collaboration

Share structured content across teams and projects.

## Markdown Tips

* Keep headings organized.
* Use lists for readability.
* Use code blocks for technical content.
* Preview Markdown before publishing.
* Keep formatting simple and consistent.
